# Product Guidelines: ILIN

## Tone & Voice
- **Professional & Direct**: All UI copy, error messages, and documentation should be concise, clear, and professional, avoiding unnecessary fluff.

## UI/UX Principles
- **Minimalist Design**: The interface should prioritize whitespace, clear visual hierarchy, and only essential elements to reduce cognitive load when querying documents.

## Accessibility
- **No Specific Focus**: For the current iteration, strict accessibility compliance (like WCAG AA) is not a primary objective; the focus is on rapid development and core functionality.

## Error Handling
- **Actionable Errors**: When issues occur (e.g., document upload failure, RAG query timeout), the system must provide clear, actionable error messages that suggest an immediate fix or next step to the user, avoiding generic or overly technical jargon.
